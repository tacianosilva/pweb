1. O que é servidor web e o que é servidor de aplicação? Qual a diferença?

Servidor de aplicação é um termo que às vezes é mesclado com um servidor web. Enquanto um servidor web manipula principalmente os protocolos HTTP, o servidor de aplicação lida com vários protocolos diferentes, incluindo, mas não limitado, a HTTP.

A tarefa principal do servidor web é exibir o conteúdo do site e o servidor de aplicação é responsável pela lógica, a interação entre o usuário e o conteúdo exibido. O servidor de aplicação é trabalhando em conjunto com o servidor web, onde um é exibido e o outro interage.

Algumas das principais diferenças em recursos do servidor web e do servidor de aplicação:

- O servidor web é projetado para fornecer conteúdo HTTP. O servidor de aplicação também pode servir conteúdo HTTP, mas não se limita apenas ao HTTP. Pode ser fornecido outro suporte ao protocolo, como RMI/RPC.

- O servidor web é projetado principalmente para fornecer conteúdo estático, embora a maioria dos servidores web tenha plug-ins para suportar linguagens de script como Perl, PHP, ASP, JSP etc., através das quais esses servidores podem gerar conteúdo HTTP dinâmico. [1]

2. Escolha uma linguagem de programação e pesquise sobre frameworks para a linguagem. Fale um pouco sobre um framework web (front-end) e sobre um framework ORM (back-end) e da linguagem escolhida.

Django e Flask são dois frameworks escritos em Python, com foco no desenvolvimento web.

Django

O Django é uma estrutura escrita em Python busca o desenvolvimento rápido, design limpo e pragmático. Com código aberto e gratuito, a principal proposta é ajudar o desenvolvedor ou desenvolvedora a pensar na aplicação.

Sua principal convenção é o DRY (Don't Repeat Yourself, em tradução livre: não seja repetitivo), que visa o máximo de proveito do código criado evitando código duplicado.

Django utiliza um padrão similar ao MVC (model-view-controller), porém aqui chamado de MTV (model-template-view), onde dividimos a aplicação em camadas ou partes, tornando o código mais organizado e legível.

Flask

O Flask é baseado na biblioteca WSGI e têm como característica a simplicidade e uma velocidade de criação superior ao Django. É considerado por muitos um pequeno framework, mas ser pequeno não significa que ele faz menos do que outras estruturas.

O Flask foi projetado como uma estrutura extensível desde o início, fornecendo tudo que é necessário para realizar as funcionalidades mais básicos, enquanto as extensões fornecem o restante.[2]

3. Pesquise e fale um pouco sobre branches e pull request usando git e github. Escreva um resumo.

Os branches (ramos) permitem que você desenvolva recursos, corrija erros ou experimente com segurança novas ideias em uma área contida do seu repositório. Você então poderá trabalhar nesse novo branch isolado das mudanças que outras pessoas estão fazendo no repositório. Aliado a isso, os administradores do repositório podem habilitar, de modo adicional, várias outras configurações de branch protegido para aplicar vários fluxos de trabalho antes que anche um br passe por um merge. [3]

As pull requests permitem que você informe outras pessoas sobre as alterações das quais você fez push para um branch em um repositório no GitHub. Após inicialização de uma pull request, você verá uma página de revisão que mostra uma visão geral de alto nível das alterações entre seu branch (o branch de comparação) e o branch base do repositório. É possível adicionar um resumo das alterações propostas, revisar as alterações feitas pelos commits, adicionar etiquetas, marcos e responsáveis, bem como fazer @menção a contribuidores individuais ou equipes. Depois que tiver criado uma pull request, você poderá fazer push dos commits branch de tópico para adicioná-los à sua pull request existente. [4]
